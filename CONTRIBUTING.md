# Contributing to Research-Repo-Template

Thank you for considering contributing to this research repository template! This document outlines the guidelines for contributing to this project.

## How to Contribute

### Reporting Issues
- Use the issue tracker to report bugs or suggest features
- Check existing issues before creating a new one
- Provide detailed information when reporting bugs (steps to reproduce, expected vs. actual behavior)

### Pull Requests
1. Fork the repository
2. Create a new branch for your feature or bugfix (`git checkout -b feature/your-feature-name`)
3. Make your changes
4. Run tests if applicable
5. Commit your changes with clear commit messages
6. Push to your branch
7. Submit a pull request

### Code Style
- Follow the existing code style and conventions
- Include comments where necessary
- Write clear, descriptive commit messages

## Directory Structure
Please maintain the repository structure:
```
├── CONTRIBUTING.md    # How to contribute
├── LICENSE
├── README.md
├── data/              # (gitignored) raw & processed files
├── docs/              # project documentation
├── notebooks/         # Jupyter notebooks
├── src/               # Python modules & scripts
└── tests/             # Unit tests
```

## Development Process
1. **Code**: Write clean, well-documented code
2. **Test**: Add tests for new features
3. **Document**: Update documentation to reflect changes

## Code of Conduct
- Be respectful and inclusive
- Provide constructive feedback
- Focus on the best outcome for the project

Thank you for your contributions!
