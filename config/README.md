# Configuration Files

This directory contains configuration files for the project.

## Usage

- Store configuration files for experiments, models, and other settings
- Use YAML, JSON, or INI formats for better readability and parsing
- Consider using template configuration files with placeholders
- Document configuration parameters in comments or separate documentation

## Best Practices

- Keep sensitive information (API keys, credentials) out of configuration files
- Use environment variables or a separate .env file (gitignored) for secrets
- Version control configuration templates, not instance-specific configurations
- Consider organizing configurations in subdirectories for complex projects
- Use descriptive names for configuration files
